# TIL とは？

TIL というのは、世界中の開発者の間で、10 年くらい前から行われている、学習記録の習慣。

→「Today I Learned の略で、「今日学んだこと」的な意味合い。

GitHub を使用して細かいアウトプットを習慣化することで、学習の過程を客観的に見える化できる。
その上、GitHub に草も生えるし、Git/ GitHub の使用に慣れることにも繋がるので、良い習慣！

**TIL（Today I Learned）の特徴：**

- 📝 **毎日の学び**: その日に学んだことを簡潔に記録
- 🌱 **小さな気づきで OK**: 完璧な記事である必要なし
- 📚 **知識の蓄積**: 自分だけの学習データベースになる
- 🔍 **後から検索可能**: 過去の学びをすぐに振り返れる
- 📝 **記事として公開**: 個人的な記録を編集して記事として発信可能

**TIL のメリット：**

1. **学習の定着**: 書くことで理解が深まる
2. **GitHub の草**: 毎日コミットで活動が可視化
3. **発信の練習**: 気軽にアウトプットの習慣がつく
4. **ポートフォリオ**: 学習姿勢の証明になる
5. **知識の整理**: 情報を構造化する力が身につく
